## Critical Decisions

### Decision 1: Interface First
Create `AeroInputProvider` in core BEFORE implementing MediaPipeProvider. This prevents rework.

### Decision 2: Test-First Within Phases
For each file, write test stubs first, then implement. Use red-green-refactor.

### Decision 3: No Real Hardware in Tests
All tests use mocks. Camera tests use `mock_mediapipe_server.py`. Keeps tests fast and CI-friendly.

### Decision 4: Submodule Strategy
Assembly uses git submodules for core and input driver. Setup via `setup_dev.py`.
