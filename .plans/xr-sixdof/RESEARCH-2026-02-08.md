## Executive Summary

As of February 2026, **XR devices offer compelling input alternatives for AeroBeat**, providing true 6DOF (six degrees of freedom) tracking that can detect punches in 3D space with high accuracy. This document covers two major XR platforms: Meta Quest 3 (inside-out tracking) and SteamVR/Valve Index (outside-in/Lighthouse tracking), along with OpenXR integration strategies for Godot 4.6.

**Key Findings:**
- Quest 3: Inside-out tracking, no external sensors needed, $500 entry point
- Valve Index: Outside-in with sub-millimeter precision, best tracking quality
- Godot 4.6 has OpenXR support via GDExtension
- 6DOF enables spatial punch detection (head, body, low zones)
- Physical safety is critical for VR boxing

## 1. Meta Quest 3 (Inside-Out Tracking)

### 1.1 Overview

**Tracking Technology:** Inside-out (cameras on headset track environment)  
**Controllers:** Touch Pro (self-tracking) or hand tracking  
**Price:** $500 USD  
**Standalone:** Yes (no PC required, optional PC VR via Link/Air Link)

### 1.2 Tracking Capabilities

| Feature | Specification | Notes |
|---------|--------------|-------|
| Position Tracking | 6DOF | Full spatial positioning |
| Rotation Tracking | 6DOF | Complete orientation |
| Tracking Volume | Guardian boundary (up to 15x15m) | User-defined play space |
| Latency | ~15-25ms | Good for fast-paced games |
| Controller Battery | 8-10 hours | Touch Pro controllers |

### 1.3 OpenXR Integration

**Godot 4.6 OpenXR Setup:**
```gdscript
extends Node3D

var xr_interface: XRInterface

func _ready():
    xr_interface = XRServer.find_interface("OpenXR")
    if xr_interface and xr_interface.is_initialized():
        get_viewport().use_xr = true
        print("OpenXR initialized successfully")
    else:
        push_warning("OpenXR not available")

func _process(_delta):
    # Get headset position and orientation
    var head_transform = XRServer.get_hmd_transform()
    var head_position = head_transform.origin
    var head_rotation = head_transform.basis.get_euler()
```

- 2 base stations (v2.0) in opposite corners

**Optimal Configuration:**
- 4 base stations for large play spaces
- Eliminates occlusion issues
- Full 360° tracking coverage

## 3. 6DOF Punch Detection

### 3.1 Spatial Punch Zones

```gdscript
enum PunchZone { HEAD, BODY, LOW }

func detect_punch_zone(hand_position: Vector3, head_position: Vector3) -> PunchZone:
    var relative_height = hand_position.y - head_position.y
    
    if relative_height > 0.2:
        return PunchZone.HEAD  # Uppercut / High punch
    elif relative_height > -0.3:
        return PunchZone.BODY  # Jab / Cross / Hook
    else:
        return PunchZone.LOW   # Body shot / Kidney punch
```

### 3.2 Velocity-Based Punch Detection

```gdscript
class_name PunchDetector6DOF
extends Node

var _last_positions: Dictionary = {}
var _last_velocities: Dictionary = {}
const PUNCH_VELOCITY_THRESHOLD: float = 2.0  # m/s
const PUNCH_ACCELERATION_THRESHOLD: float = 5.0  # m/s²

signal punch_detected(hand: String, punch_type: String, zone: int, power: float)

func track_controller(hand: String, position: Vector3, delta: float):
    if not _last_positions.has(hand):
        _last_positions[hand] = position
        _last_velocities[hand] = Vector3.ZERO
        return
    
    # Calculate velocity
    var velocity = (position - _last_positions[hand]) / delta
    var acceleration = (velocity - _last_velocities[hand]) / delta
    
    # Detect punch
    var speed = velocity.length()
    var accel = acceleration.length()
    
    if speed > PUNCH_VELOCITY_THRESHOLD and accel > PUNCH_ACCELERATION_THRESHOLD:
        var punch_type = classify_punch(velocity, _last_velocities[hand])
        var zone = detect_punch_zone(position, get_head_position())
        var power = clamp(speed / 5.0, 0.3, 1.0)
        
        punch_detected.emit(hand, punch_type, zone, power)
    
    _last_positions[hand] = position
    _last_velocities[hand] = velocity

func classify_punch(velocity: Vector3, prev_velocity: Vector3) -> String:
    # Classify based on velocity direction
    var horizontal = Vector2(velocity.x, velocity.z).length()
    var vertical = velocity.y
    
    if vertical > horizontal * 1.5:
        return "uppercut"
    elif abs(velocity.x) > abs(velocity.z):
        return "hook"
    else:
        return "jab"
```

## 5. Physical Safety in VR Boxing

### 5.1 Critical Safety Considerations

| Risk | Mitigation |
|------|-----------|
| **Punching real objects** | Guardian boundary, 1m+ safety buffer |
| **Tripping/falling** | Guardian floor tracking, seated mode option |
| **Hitting other people** | Play space isolation, passthrough warning |
| **Motion sickness** | Comfort settings, vignetting, snap turning |
| **Eye strain** | IPD adjustment, brightness controls |
| **Wrist strain** | Proper controller grip, wrist straps |

### 5.2 Guardian Integration

```gdscript
# Safety warning system
extends Node

@export var safety_buffer: float = 1.0  # meters from boundary

func _ready():
    XRServer.boundary_entered.connect(_on_boundary_entered)
    XRServer.boundary_exited.connect(_on_boundary_exited)

func _on_boundary_entered():
    # Pause game, show warning
    Events.pause_game.emit()
    UI.show_safety_warning("You've reached the play area boundary!")
    
    # Optional: Show passthrough
    enable_passthrough(true)

func _on_boundary_exited():
    # Resume when back in safe zone
    Events.resume_game.emit()
    UI.hide_safety_warning()
    enable_passthrough(false)
```

### 5.3 Comfort Settings

```gdscript
# VR comfort options
@export_group("Comfort Settings")
@export var vignette_on_movement: bool = true
@export var snap_turning: bool = false
@export var snap_turn_angle: float = 45.0
@export var smooth_turn_speed: float = 90.0  # degrees/second

func handle_turning(input: Vector2):
    if snap_turning:
        if abs(input.x) > 0.5:
            rotate_y(deg_to_rad(snap_turn_angle * sign(input.x)))
    else:
        rotate_y(deg_to_rad(input.x * smooth_turn_speed * get_process_delta_time()))
```

## 6. Haptic Feedback

### 6.1 Controller Vibration Events

| Event | Pattern | Intensity |
|-------|---------|-----------|
| Punch Landed | Short pulse (50ms) | 0.6 |
| Punch Blocked | Double pulse | 0.4 |
| Heavy Punch | Long pulse (150ms) | 0.9 |
| Player Hit | Strong pulse (200ms) | 0.8 |
| Low Stamina | Pulsing heartbeat | 0.3 |
| Victory | Celebration pattern | 0.7 |

```gdscript
func trigger_haptic(event_type: String, controller_id: int):
    var pattern = get_haptic_pattern(event_type)
    
    for pulse in pattern:
        XRServer.get_controller(controller_id).set_haptic_pulse(
            pulse.frequency_hz,
            pulse.amplitude,
            pulse.duration_sec
        )
        await get_tree().create_timer(pulse.delay).timeout

func get_haptic_pattern(event: String) -> Array:
    match event:
        "punch_landed":
            return [{"freq": 160, "amp": 0.6, "dur": 0.05, "delay": 0}]
        "heavy_punch":
            return [{"freq": 320, "amp": 0.9, "dur": 0.15, "delay": 0}]
        "player_hit":
            return [
                {"freq": 160, "amp": 0.8, "dur": 0.1, "delay": 0},
                {"freq": 160, "amp": 0.8, "dur": 0.1, "delay": 0.05}
            ]
    return []
```

## 10. Conclusion & Recommendations

### Summary

1. **XR is viable for AeroBeat** — 6DOF tracking enables true spatial boxing
2. **Quest 3 offers best value** — wireless, standalone, good tracking
3. **Index offers best quality** — sub-millimeter precision, finger tracking
4. **Safety is critical** — Guardian/boundary integration required
5. **Timeline: 6-8 weeks** for full XR support

### Final Recommendation

**Implement XR support in phases:**

1. **Phase 1:** Quest 3 support (largest user base, standalone convenience)
2. **Phase 2:** SteamVR support (best tracking quality for enthusiasts)
3. **Phase 3:** Hand tracking fallback (controller-free option)

This approach provides:
- Broad platform coverage
- Optimal user experience per platform
- Future-proof OpenXR architecture
- Strong safety systems
